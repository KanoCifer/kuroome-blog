# 问题与需求 Backlog

> 随手记录发现的问题和待开发的需求。定期 review，可行的转入-issue 或列入计划。

---

## 格式说明

每条记录使用以下结构（不必每项都填，按需取舍）：

```markdown
### [P/F] 简短标题

- **类型**: 问题(Bug) / 功能需求(Feature) / 优化(Improvement) / 技术债(Tech-debt)
- **优先级**: P0 紧急 / P1 高 / P2 中 / P3 低
- **范围**: 前端-Vue / 前端-React / 后端-Python / 后端-Go / 通用
- **状态**: 待评估 / 待排期 / 进行中 / 已搁置
- **描述**: 一句话说清问题或需求
- **详情**: 补充信息——复现步骤、期望行为、参考链接、初步方案等
```

- **[P]** = 问题（已有现象）
- **[F]** = 功能需求（想做但还没有）
