# 核心模块文档

本文档描述 ReadingList 后端的 11 个核心模块。所有模块统一挂载在 `/api/v1` 命名空间下。

## 1. 模块概览

| 模块 | 基础路由 | 说明 |
| --- | --- | --- |
| 认证 | `/api/v1/auth` | 登录、注册、JWT、Passkey、OAuth |
| 书籍 | `/api/v1/books` | 书架、书籍信息、阅读进度 |
| 博客 | `/api/v1/blog` | 文章、分类、评论、发布 |
| 留言板 | `/api/v1/messages` | 访客留言与管理回复 |
| 微信读书 | `/api/v1/weread` | 微信读书导入与同步 |
| RSS | `/api/v1/rss` | 订阅源管理、抓取、聚合 |
| 管理员 | `/api/v1/admin` | 内容审核、后台管理、权限操作 |
| AI | `/api/v1/ai` | 摘要、对话、内容增强 |
| 待办 | `/api/v1/todos` | 任务管理与个人事项 |
| 监控 | `/api/v1/monitor` | 系统状态、统计、运行指标 |
| 公开 API | `/api/v1/public` | 对外公开的通用接口 |

## 2. 每个模块的详细说明

### 2.1 认证模块

- 负责用户注册、登录、退出、刷新令牌
- 支持 Passkey 凭证与第三方登录
- 统一处理 Cookie、JWT、CSRF 等安全机制

### 2.2 书籍模块

- 管理书籍增删改查
- 维护用户书架与阅读进度
- 支持从外部来源导入书籍元数据

### 2.3 博客模块

- 管理文章发布、编辑、删除
- 处理分类、置顶、评论、审核
- 支持 MongoDB 文档型存储

### 2.4 留言板模块

- 提供访客留言功能
- 支持留言审核、管理员回复
- 用于站点互动与反馈收集

### 2.5 微信读书模块

- 同步微信读书书架、笔记、划线内容
- 将外部数据清洗后写入本地结构
- 适合做读书记录整合

### 2.6 RSS 模块

- 管理 RSS 订阅源
- 定时抓取 RSS 内容并解析为文章
- 支持文章去重、缓存与已读状态

### 2.7 管理员模块

- 提供后台审核与管理能力
- 负责内容状态控制、用户管理、系统配置
- 一般只允许管理员角色访问

### 2.8 AI 模块

- 为书籍或文章生成摘要
- 提供内容润色、问答等能力
- 通常结合 Redis 做结果缓存

### 2.9 待办模块

- 管理个人待办事项
- 适合轻量任务记录和状态跟踪
- 与用户账号强绑定

### 2.10 监控模块

- 输出系统健康状态、访问统计、任务状态
- 用于运维和后台可视化
- 也可承载访客统计信息

### 2.11 公开 API 模块

- 为外部页面或第三方提供无需复杂认证的公共接口
- 适合做轻量展示、基础信息查询、公开内容读取

## 3. 关键业务逻辑链

### 3.1 书籍业务链

用户登录 → 添加书籍 / 导入书籍 → 写入书架 → 更新阅读进度 → 生成统计数据。

### 3.2 博客业务链

管理员发布文章 → 分类入库 → 用户浏览 → 评论/留言 → 审核与展示。

### 3.3 RSS 业务链

用户订阅 RSS → 定时抓取源站 → 解析条目 → 去重入库 → 供前端阅读。

### 3.4 微信读书同步链

提交同步请求 → 拉取外部数据 → 解析书名/笔记/进度 → 合并书架数据 → 更新本地库。

## 4. 统一响应格式

```json
{
  "code": 0,
  "message": "success",
  "data": {},
  "timestamp": 1710000000000
}
```

- `code`：业务状态码
- `message`：人类可读提示
- `data`：实际返回数据
- `timestamp`：服务端时间戳

## 5. 权限控制原则

- **游客**：可访问公开 API、博客列表、留言板展示、监控公开信息
- **登录用户**：可使用书籍、RSS、待办、AI 等个人功能
- **管理员**：可访问后台审核、内容管理、系统配置与敏感监控信息
